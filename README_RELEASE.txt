# HA Lighting Compilers — инструкция наладчика

Версию сборки видно в шапке лаунчера. Называйте её, когда с чем-то обращаетесь.

---

## Что внутри архива

- `launcher.exe` — окно, из которого всё запускается
- `scripts/` — сами генераторы. ОБЯЗАТЕЛЬНЫ: лаунчер их только запускает
- `templates/` — шаблоны карточек Lovelace, можно править
- `data/` — сюда кладёте свой Excel, сюда же ложится результат
- `requirements.txt` — зависимости Python
- `THIRD-PARTY-LICENSES.md` — лицензии стороннего материала

⚠ `launcher.exe` НЕ содержит генераторы внутри. Он запускает `scripts/*.py`
через Python из `.venv`. Поэтому структуру папок менять нельзя.

---

## Установка (один раз)

1. Поставьте Python 3.10–3.12 с python.org, галочка **Add Python to PATH**.
   Проверка: `py --version`

2. Распакуйте архив, например в `C:\Projects\ha-lighting-compilers`

3. Откройте папку, наберите `powershell` в адресной строке проводника, Enter.

4. Создайте окружение и поставьте зависимости:

   ```
   py -m venv .venv
   .venv\Scripts\python.exe -m pip install -r requirements.txt
   ```

   ⚠ Именно `.venv\Scripts\python.exe -m pip`, а не просто `pip`: иначе пакеты
   уедут не в то окружение, и лаунчер их не найдёт.

## Работа

Двойной клик по `launcher.exe`. В шапке — имя инструмента и версия.

### Поля вверху

| Поле | Что это |
|---|---|
| **Project Root** | папка, куда распаковали архив |
| **Excel File Path** | ваша таблица в `data/` |
| **Дашборд** | `url_path` дашборда на объекте (например `dashboard-tets`). Обязан содержать дефис — требование Home Assistant |
| **Объект** | имя в шапке Главной страницы (например «Колледж Химки») |

⚠ «Дашборд» и «Объект» читает **генерация карточек**, а не деплой. Поменяли имя
— прогоните `[ +] LOVELACE CARDS` заново, иначе на дашборд уедет старое.

### Кнопка `[ ▶] BUILD ALL` — девять шагов подряд

```
[01] VALIDATE EXCEL      проверка таблицы. Ошибки — пайплайн стоит
[02] NORMALIZE EXCEL     Excel -> parquet, дальше читается только он
[03] LIGHTS GROUPS       группы света по зонам
[04] GENERAL GROUPS      общая группа на помещение
[05] FLOOR GROUPS        группы этажей и техпомещений
[06] AREAS & FLOORS      файл-задание на пространства и этажи
[07] HELPERS             вспомогательные объекты одним пакетом
[08] SCRIPTS             клоны скриптов по единицам обслуживания
[09] AUTOMATIONS         автоматизации + blueprint'ы
```

Первым идёт проверка таблицы: на битой таблице сборка останавливается сразу, а
не показывает ошибки уже в Home Assistant.

**Build All работает офлайн.** В Home Assistant он не пишет ничего.

### `[ +] LOVELACE CARDS` — карточки

В Build All **не входит**: карточки нужны не всегда, а для залов и рекреаций на
Home Assistant должны стоять карты `mushroom` и `card_mod` из HACS.

Кнопка собирает views дашборда в `data/lovelace/`. Экспорт в txt с копипастом
руками — метод v1, его больше нет: карточки уезжают деплоем.

### `[ ↑] DEPLOY → HOME ASSISTANT` — единственный шаг, который трогает объект

Открывается отдельное окно: галочки «что отправить», параметры SSH и Home
Assistant, кнопки **«Показать план (dry-run)»** и **«Отправить на HA (LIVE)»**.

Порядок на объекте:

1. `Build All`
2. `[ +] LOVELACE CARDS`
3. **Снимок дашборда**: `.venv\Scripts\python.exe scripts\backup_dashboard.py`
   Запись конфига дашборда перезаписывает его ЦЕЛИКОМ — снимок обязателен.
4. **Dry-run.** Покажет, что поедет, ничего не отправляя.
5. **LIVE.**

⚠ **Рестарт Home Assistant деплой НЕ делает.** Файлы приедут, но не применятся,
пока не перезапустите вручную. Карточек и пространств это не касается — они
применяются сразу.

⚠ Для карточек нужен токен **АДМИНИСТРАТОРА**: запись конфига дашборда обычному
пользователю запрещена.

## Частые ошибки

**«python.exe не найден»** — не создан `.venv`. Вернитесь к установке.

**Кнопка нажимается, ничего не происходит** — проверьте, что
`.venv\Scripts\python.exe` существует и Project Root указывает на папку с
`scripts/`.

**Деплой: «invalid auth»** — токен не тот или не админский.

**Деплой карточек молчит, а на дашборде старое** — не прогнали
`[ +] LOVELACE CARDS` после правки полей «Объект»/«Дашборд». Деплой ничего не
генерирует, он льёт готовые файлы с диска.

**После деплоя пакет не загрузился, в логе HA «duplicate key»** — на объекте
уже есть помощник с таким же именем, заведённый руками. Home Assistant не
сливает его с нашим пакетом, а роняет пакет целиком. Удалите ручного.

---

## Чего инструмент НЕ делает

- **Не перезапускает** Home Assistant.
- **Не собирает JSON Zone Manager.** Без него автоматизация получит
  `found: false`, и свет включаться не будет. Это отдельная ручная
  конфигурация.
- **Не трогает то, что вы сделали руками.** На дашборде наши разделы помечены
  префиксом `zm-`; всё остальное деплой сохраняет как есть.

# 🏫 HA College Lighting — генераторы конфигурации

Проект содержит набор Python-скриптов для генерации конфигураций и Lovelace-карточек
Home Assistant OS (HAOS) на основе единого источника данных (Excel).

Основная цель:
- автоматизировать создание групп света,
- обеспечить единый нейминг сущностей,
- генерировать визуальные карточки управления освещением,
- снизить количество ручной YAML-работы.

## 🧠 Общая логика

- **Excel (.xlsx)** — единый источник данных по помещениям, группам света и датчикам.
- Все генераторы читают данные из таблицы и создают **текстовые/YAML-файлы**.
- Генерация идёт по принципу *компилятора*, а не YAML-сериализации:
  - шаблоны не ломаются,
  - отступы и формат сохраняются,
  - логика контролируется regex-заменами.

Подробные правила и допущения см.  
👉 `docs/internal/architecture_rules.md`

## 🧩 Скрипты

### 1️⃣ `generate_lights_groups.py`
**Назначение:**  
Создание базовых групп светильников (подгрупп помещений).

**Что делает:**
- читает Excel,
- группирует светильники по `group_id`,
- создаёт YAML с `platform: group`,
- формирует сущности вида:
light.403_0
light.403_1


**Результат:**
- `lights_group.yaml`

---

### 2️⃣ `generate_general_groups.py`
**Назначение:**  
Создание *общей группы помещения* на основе подгрупп.

**Что делает:**
- использует уже созданные группы (`403_0`, `403_1`, …),
- формирует общую группу: light.403_obshchii_kabinet_medits
- корректно транслитерирует имена с учётом исключений.

**Результат:**
- `lights_general_groups.yaml`

---

### 3️⃣ `generate_floor_groups.py`
**Назначение:**  
Создание групп по этажам.

**Что делает:**
- берёт **общие группы помещений**,
- объединяет их по этажу (100 → 1 этаж, 200 → 2 этаж и т.д.),
- создаёт группы вида:
light.floor_1_all
light.floor_2_all

**Результат:**
- `lights_floor_groups.yaml`

---

### 4️⃣ `generate_lovelace_cards.py`
**Назначение:**  
Генерация визуальных Lovelace-карточек для управления освещением.

**Особенности:**
- использует **глоссарий карточек** (`Глоссарий карточек.txt`);
- карточки отличаются по:
- типу помещения (коридор, лестница, кабинет, тамбур, с/у),
- количеству групп света;
- шаблон выбирается:
1. по точному совпадению `(тип, кол-во групп)`,
2. либо по ближайшему варианту (без автомасштабирования).

**Что подставляется автоматически:**
- entity_id света,
- датчики движения / освещённости,
- заголовки и имена карточек,
- корректные номера помещений.

**Результат:**
- `lovelace_cards_generated.txt`  
(карточки размечены блоками `{{ ... }}` для удобного копирования)

## 🗂 Глоссарий карточек

Файл: data/Глоссарий карточек.txt

- свободный текстовый формат;
- YAML-карточки заключены между:
{{
...
}}
- внутри могут использоваться Jinja-выражения (`{{ }}`), поэтому парсинг ведётся
**по строкам-границам**, а не regex-блоками.

## ⚙️ Пути и запуск

Все скрипты используют пути **от корня проекта**:

```python
BASE_DIR = Path(__file__).resolve().parent.parent
```
Это позволяет:
- запускать скрипты из scripts/,
- не зависеть от рабочей директории PyCharm,
- безопасно работать после переименования проекта.

Запуск (пример):
```bash
python scripts/generate_lovelace_cards.py
```

🧾 Что намеренно НЕ хранится в Git

- .venv/
- реальные .xlsx с рабочими данными,
- сгенерированные YAML/TXT файлы,
- черновики и архивы (old group/)

См. .gitignore.
